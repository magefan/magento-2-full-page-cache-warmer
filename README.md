[Magento 2 Full Page Cache Warmer](https://magefan.com/magento-2-full-page-cache-warmer)  is a fully automated solution that warms up the cache for every page to deliver them faster. It allows you to create cache warming combinations, exclude pages and user agents' cache warming, monitor the entire process and more. 

Improve your website loading speed with zero effort. Deliver the most recent version of the page to your customers faster since Magento Cache Warmer always keeps it warm.

## Features
### Automate Cache Warming Process

 When flushed too often, full page cache can lead to delays in loading speed. All because the page needs to be loaded from scratch instead of loading from cache. But not any more. 

Magento 2 Full Page Cache Warmer will imitate user behaviour to keep the cache warm for all of the most popular pages. 

Working in the background, this solution causes no interruptions in the backend or frontend actions. 

Continue working on your store tasks. We’ll keep your cache updated automatically, at all times.

<p align="center">
  <img width="860" height="428" src="https://cm.magefan.com/wysiwyg/products/automate-magento-cache.png">
</p>

### Keep Cache Warm for Most Popular Pages

When visitors come to products or categories ready to purchase, the last thing they need is to wait for the page to load. We just can’t let that happen.

Magento 2 Full Page Cache Warmer rates pages based on the number of visits and adds them at the top of the cache warming queue. This way the most popular pages are always delivered the fastest. 

Additionally, you can create warmup combinations based on customer groups, currencies and store views where you need to warm up the full page cache.

<p align="center">
  <img width="860" height="428" src="https://cm.magefan.com/wysiwyg/products/cache-warming-priorities-magento.png">
</p>

### Exclude Pages and User Agents from Cache Warming

The top task is to deliver the most popular pages to customers first. If you dedicate all your resources to all pages at once, some popular ones might be missed. 

That scenario is completely covered by the Magento 2 Cache Warmer. 

You can exclude pages and page types that aren’t updated that often from the cache warming to prioritize your resources. 

Besides, you can exclude different user agents from cache warming.

<p align="center">
  <img width="860" height="428" src="https://cm.magefan.com/wysiwyg/products/exclude-pages-from-cache-warmin-magento.png">
</p>

### Choose Performance Preset for Your Server

To warm up the cache in Magento, you need to be sure your server can handle it. But you don’t have to worry about that with the Magefan Cache Warmer. 

It’s already set up to use the Medium mode for the cache warming. If you want to change it, you can choose high, low or custom intensity.

The latter allows you to customize the cache warming performance completely. 

Choose the number of concurrent requests to your store and the cache warming job schedule. Set job max run time and delay between requests.

<p align="center">
  <img width="800" height="500" src="https://cm.magefan.com/wysiwyg/products/cache-warmer-performace-presets.png">
</p>

### Keep Track of all Cache Warming Processes

Once you install and enable the Magento 2 Full Page Cache Warmer you don’t have to do anything. Except monitor how the extension warms up the cache for your pages.

Check all cached and non-cached pages in one place. View cache warming coverage and full-page cache history.

<p align="center">
  <img width="800" height="500" src="https://cm.magefan.com/wysiwyg/products/magento-cache-coverage-reports.png">
</p>

### Monitor Cache Flushes

When the cache is flushed, page content needs to be loaded from scratch which causes a delay. So it’s not a favourable scenario. 

While the Magento 2 Full Page Cache Warmer will keep the cache warm, it can’t control the cache flushes.

However, it allows you to figure out why the cache was flushed in the first place, who flushed it and when the cache flushes report.

</p>
